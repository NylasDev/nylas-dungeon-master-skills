# Sourcebook hygiene

This repository teaches DM craft. It should not mirror, summarize, or replace commercial rulebooks, adventures, campaign settings, or private books a user provides.

Use this reference when adding public docs, examples, templates, or skill text that was inspired by a rulebook, campaign guide, SRD, actual play, or user-provided source.

## Allowed in public repo content

- Original procedures, checklists, templates, and examples.
- High-level tabletop design principles: agency, spotlight, consequences, pacing, safety, hidden information, faction pressure.
- User-owned DM philosophy and original campaign craft.
- Original names, factions, places, monsters, scenes, and sample campaigns.
- SRD-safe rules concepts when the license permits reuse and attribution is handled correctly.
- Short references to broad genres or play styles, such as heroic fantasy, science fantasy, dungeon crawl, political intrigue, or survival horror.

## Do not publish

- Copied rulebook prose.
- Class, spell, item, monster, feat, ancestry/species, or background rules text from a protected source.
- Stat blocks or tables copied from a book or website.
- Boxed text, room descriptions, adventure beats, maps, encounter keys, or faction prose from published adventures.
- Setting-specific names, locations, organizations, NPCs, gods, pantheons, or lore unless the license clearly allows public reuse.
- Long summaries that let someone avoid owning the source.
- Private campaign secrets Nylas or a user has not approved for public release.

## Private play versus public publishing

A user may provide a private book, PDF, campaign note, or rules excerpt so the agent can help at their table. Treat that as private context.

For private play support, the agent can:

- Use the material to answer the user's own campaign questions.
- Help the user apply rules at their table.
- Summarize their own notes for session use.
- Adapt provided material into private prep.

For public repository content, the agent must:

- Extract general methods instead of copying expression.
- Replace protected names with original examples.
- Rewrite examples from scratch.
- Keep attribution and license notes clean when using permissive sources.

## SRD use

If SRD material is used, check the current license first. Keep SRD-derived text minimal and properly attributed. Prefer using the SRD as a rules compatibility reference, not as the voice or body of the skill.

Safer pattern:

> In a d20 fantasy game, ask for an ability check when the action is uncertain and failure would matter.

Risky pattern:

> Copying a rule paragraph, spell entry, class feature, item description, or monster block.

## Public examples

Public examples should use original material. Invent the village, NPCs, threats, relics, factions, gods, ships, ruins, and monsters. Avoid recognizable protected setting content.

Good public example:

- A salt-mining town built over a singing cavern.
- A failed sun cult trying to relight a dead lighthouse.
- A rival cartographer hiding a map inside a tattoo.

Bad public example:

- A scene lifted from a published adventure with names changed.
- A famous setting location with serial numbers filed off.
- A monster stat block copied from an online rules compendium.

## Contributor checklist

Before committing public content:

- [ ] The content is original or license-safe.
- [ ] No copied tables, stat blocks, boxed text, or sourcebook prose appear.
- [ ] Protected setting names are absent unless used only in a legal/source discussion.
- [ ] User-provided private material stays private.
- [ ] The result teaches a method, not a book.
